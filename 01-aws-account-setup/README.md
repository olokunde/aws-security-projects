## Introducing Today's Project!

Every cloud environment I'll ever secure starts with this: an account done right. In this project, I'm setting up my AWS account the way a security engineer would on day one — Free Tier billing alerts configured before I build anything, and MFA locked onto the root account so a password alone can never open the door. It's the foundation for the security projects ahead: encryption, threat detection, and network monitoring.

### Key tools and concepts

The key tools I used include the AWS Console, Billing Preferences, and an authenticator app for MFA. Key concepts I learnt include the root user — the identity with full access to the entire account — the Free Tier, and setting usage alerts so AWS warns me by email before I exceed a limit.

### Challenges and wins

This project took me approximately 35 minutes. The most challenging part was when I first got an error while trying to sign up.

## Creating My AWS Account

 I'm signing up for my AWS account: verifying my email and creating a strong root user password. Since the root user holds unrestricted access to the entire account, securing it properly is the first real security decision I'm making on this platform.

### Understanding the root user

I learned that a root user is the main account owner with full access to everything in AWS. A strong password matters because if it's hacked, an attacker controls everything.

## Setting Up Billing and Payment

In this step, I'm going to choose the Free plan, entering my contact information, and adding a payment method — which AWS requires to verify my identity, not to charge me.

### Why AWS requires a payment method

I learned that AWS requires a payment method to verify that I'm a real person, and so they have a way to bill me only if my usage ever goes beyond the Free Tier limits.

## Completing Account Activation

In this step, I'm going to be confirming my identity with an SMS verification code and selecting the Basic support plan

### Identity verification and support plan

I verified my identity through SMS verification. The Basic support plan include documentation, forums, and Trusted Advisor checks.

## Enabling Free Tier Alerts

In this step, I'm going to log into the AWS Console, navigate to Billing Preferences, and enable Free Tier usage alerts so AWS warns me by email before I ever exceed a Free Tier limit.
